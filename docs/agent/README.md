> ## 维护规则
>
> 1. **何时更新**：当代码发生结构性变更时（新增/删除文件、修改核心接口、重构架构），Agent 应主动更新对应文档
> 2. **何时无需更新**：纯 UI 微调、文案修改、Bug 修复不影响文档准确性时，无需更新
> 3. **更新原则**：保持文档与代码一致，宁可短暂过时也不要写入不确定的内容
> 4. **格式约束**：使用中文撰写，Markdown 格式，每个文件聚焦一个主题
>
